# Site Architecture

This repository is a compact Hugo site built around a pinned `hugo-bearblog` theme and local overrides. The project is intentionally close to static HTML: Markdown content, Hugo templates, inline CSS, and copied static assets.

## Rendering Model

```text
content/*.md
  -> layouts/_default/*.html or layouts/index.html
  -> layouts/partials/*.html
  -> static/*
  -> public/
```

`public/` is generated by Hugo and should not be edited by hand.

## Configuration

- `baseURL` is `https://meteoryee.github.io`.
- `theme` is `hugo-bearblog`.
- `enableGitInfo = true`, so page last-modified output can depend on Git history.
- `disableKinds = ["taxonomy"]`, so taxonomy pages are intentionally not generated.
- Blog permalinks are flattened with `blog = "/:slug/"`.

Hugo currently emits deprecation warnings for `languageCode` and `.Site.LanguageCode`. They do not break the build, but a future cleanup can migrate config to `locale` and templates to `.Site.Language.Locale`.

## Directory Roles

- `content/_index.md`: home page content.
- `content/blog/`: personal essays and dated posts. URLs are flattened to the site root.
- `content/tech/`: technical notes. URLs stay under `/tech/`.
- `content/gallery.md`: top-level gallery page, with images stored under `static/images/gallery/`.
- `content/resume.md`: top-level resume page.
- `content/hidden/`: unlisted but published pages.
- `archetypes/`: templates for `hugo new`.
- `layouts/`: effective rendering layer for the site.
- `static/`: copied as-is to the output root.
- `themes/hugo-bearblog/`: upstream theme baseline.

## Template Overrides

The following local files differ from the upstream theme and should be treated as intentional site behavior:

- `layouts/_default/baseof.html`: page shell, metadata, style partial, header/footer, global edited date.
- `layouts/_default/list.html`: section list rendering.
- `layouts/_default/single.html`: single page rendering, table of contents, tags.
- `layouts/index.html`: home page rendering with current-date updated line.
- `layouts/partials/nav.html`: main menu plus a hand-added Blog link.
- `layouts/partials/footer.html`: local copyright text.
- `layouts/partials/style.html`: main inline CSS.

The main CSS lives in `layouts/partials/style.html`. `static/css/styles.css` is present but not referenced by the current templates.

## Content Model

Standard front matter is TOML delimited by `+++`.

Typical page:

```toml
+++
title = "Page Title"
menu = "main"
+++
```

Typical dated post:

```toml
+++
title = "Post Title"
date = "2026-02-02T23:29:25+08:00"
description = ""
tags = []
+++
```

Use `menu = "main"` only for top-level navigation pages. Avoid adding `menu` to normal blog posts or tech notes.

## AI-Native Maintenance Contract

This site is optimized for small, reviewable changes:

- Keep architecture static-first and Hugo-native.
- Record project conventions in `AGENTS.md` before relying on tool-specific prompt memory.
- Use `static/llms.txt` as the public, concise LLM-facing site map.
- Avoid duplicating instructions across many vendor-specific files unless a tool actually needs one.
- After changing templates, config, content routing, or public AI-facing files, run `hugo --gc --minify`.
